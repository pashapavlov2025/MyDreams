# MyDreams - Personal Finance & Asset Tracker

## Vision
Персональное веб-приложение (PWA) для отслеживания всех активов и обязательств с минимальными усилиями.
Подход: **snapshot-based** — раз в месяц (или когда хочется) обновляешь балансы, приложение строит историю.
Отдельный модуль для инвестиционных проектов (вилла на Бали и подобные) с полным P&L.

## Tech Stack
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **UI**: Tailwind CSS, мобильный дизайн (iOS-like)
- **Data**: IndexedDB (Dexie.js v4) — клиентское хранилище
- **Charts**: Recharts (area charts, pie charts)
- **Currency rates**: open.er-api.com (фиат) + CoinGecko (крипто)
- **Hosting**: Vercel
- **Security**: PIN-код (SHA-256, sessionStorage)

## Ключевые решения
- Web вместо iOS — доступ с любого устройства, быстрый деплой через Vercel
- IndexedDB (Dexie.js) — работает offline, без бэкенда на старте
- Snapshot-подход: каждая запись = "на эту дату баланс такой-то"
- PWA: можно установить на Home Screen

### ФАЗА 6: Доступ для близких («что если со мной что-то случится») 📋
**Цель**: чтобы жена смогла найти все активы, если меня не будет рядом.
Не пароли — карта: где что лежит и к кому идти.

**Постановка от владельца (июль 2026)**: «жена постоянно спрашивает, как она
найдёт все мои аккаунты. Почему бы не хранить базовую информацию рядом с
аккаунтами — отправлю ей бэкап, она поднимет у себя».

#### ⚠️ Граница, которую нельзя переходить

Это **не менеджер паролей**. В хранилище не должно попадать ничего, что даёт
прямой доступ к деньгам:

- ❌ пароли, PIN-коды, коды из СМС
- ❌ seed-фразы и приватные ключи (Binance, холодные кошельки)
- ❌ полные номера карт и счетов, CVV

Причина: бэкап — **обычный незашифрованный JSON**, который ездит по AirDrop и
лежит в «Файлах». PIN в приложении — 4 цифры, SHA-256 в localStorage, от
серьёзной атаки не защищает. Один утёкший файл = потеря всего.

Что хранить можно — **указатели**:
- В каком банке/брокере счёт, в какой стране, тип счёта
- Где лежат документы (папка, сейф, у нотариуса)
- К кому обращаться: юрист, бухгалтер, менеджер в банке
- Оформлен ли наследник/бенефициар по этому счёту
- Свободный комментарий «как это найти»

- Отдельная сущность «инструкция для близких» или просто поля у счетов?
- Нужен ли отдельный профиль-«досье», чтобы жена импортировала его отдельно

## Приоритеты
- **Простота ввода данных** > красота UI
- **Snapshot-подход** > детальный учёт транзакций
- **Работает** > идеально спроектировано
- Итеративный подход: сначала работает, потом красиво
